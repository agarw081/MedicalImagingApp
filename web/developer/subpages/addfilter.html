Adding a new filter

Adding a filter is very similar to adding a tool. In this instance we will define each instance of the filter in the final application they will be used in, but aside from that it is the same.

 

1.    Create a .h file for the filter you wish to create

2.    Define the .h file in the file

3.    Inherit the filter class

4.    At this point you must choose to implement a kernel-based filter or a pixel-by-pixel based filter

a.     A pixel by pixel based filter has its own override applyFilter function where you can make the changes that determine the effect of the filter

b.    A kernel-based filter implements the applyFilter function from the filter class.

5.    Once you’ve deiced on your filter type, create the corresponding .cpp file to the .h file you made earlier.

6.    Include filter.h if pixel by pixel or filter.h and kernel.h if kernel

7.    If pixel-by-pixel you’ll have to implement you own applyFilter method that overrides filter.h’s method. To do this you can start with filter.h’s implementation or copy a version from another pixel-by-pixel filter.

8.    Make your adjustments to the pixel manipulating lines to create your desired effect

9.    If you are making a kernel based filter, all you have to do is generate your kernel and then pass it into appyFilter in the Filter class

10. To generate the kernel use the various functions in the kernel class

11. Once you have you filter .cpp and .h files complete, move on to the filter factory class.

12. Include your filters .h file here and add a case in the createFilter function to build it.

13. Finally, go to your program, like photoshop or medimage, and create an instance of your filter built through the factory. Then use this instance to call the applyFilter method with the proper parameters passed in.
